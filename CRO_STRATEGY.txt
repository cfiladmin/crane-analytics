■ 診断サマリー
現在ユーザー0人の根本原因は「価値を見せる前に登録を要求している」こと。
URLを開いた瞬間にログイン画面が出る設計は、CVRを最大90%以上損失させる。
まず「価値を見せる→信頼する→行動する」の順番に設計を変えることが最優先。

STEP 1：ファネル構造の現状と問題
================================================================

【現在のファネル（壊れている）】

認知（SNS/口コミ）
  ↓
URL を開く
  ↓
【即ログイン画面】← ここで90%以上離脱
  ↓
アプリ使用

【あるべきファネル】

認知（SNS/口コミ）
  ↓
ランディングページで価値確認（30秒）
  ↓
「使ってみたい」と思う
  ↓
Googleログイン（納得済み）
  ↓
初回プレイ記録
  ↓
継続使用・口コミ


【ブロックされた動脈（最大の問題箇所）】

問題1：ランディングページが存在しない（最重大）
  → URLを開いてもアプリが何をするものか全くわからない
  → クレゲをやる人が「面白そう」と思う前にログインを要求
  → 離脱率：推定95%以上

問題2：ログイン前に価値がゼロ見える
  → スクリーンショットも説明文も何もない
  → 「知らないアプリにGoogleアカウントを渡す」という心理的ハードル
  → 離脱率：さらに悪化

STEP 2：O/CO テーブル（異議→反論）
================================================================

【ターゲットが感じる5大異議と反論】

┌────────────────────────────────────────────────────────────┐
│ 異議（Objection）          │ 反論（Counter-Objection）      │
├────────────────────────────────────────────────────────────┤
│ 1. これ何のアプリ？        │ 「クレゲに何円使ったか・あと  │
│    何が嬉しいの？          │  何手で取れるかが数字でわかる」│
│                            │  → スクショ3枚で30秒説明      │
├────────────────────────────────────────────────────────────┤
│ 2. Googleログインが怖い    │ 「メール送信・投稿は一切なし。 │
│    何かされそう            │  ログインはデータ保存のためだけ」│
│                            │  → Firebaseの説明を1行追加    │
├────────────────────────────────────────────────────────────┤
│ 3. 自分には難しそう        │ 「記録はタップ1回。動きを選ぶ  │
│    使い方がわからない      │  だけ。計算は自動」            │
│                            │  → GIF/動画でデモを見せる     │
├────────────────────────────────────────────────────────────┤
│ 4. 無料？後で課金される？  │ 「完全無料。課金要素なし」     │
│                            │  → ファーストビューに明記     │
├────────────────────────────────────────────────────────────┤
│ 5. 本当に役立つの？        │ 「損切りラインを超えたら即アラ │
│    机上の空論では？        │  ート。メルカリ相場と自動比較」│
│                            │  → 具体的な数字で示す         │
└────────────────────────────────────────────────────────────┘

【フェーズ1：今週中（ユーザー0→初期ユーザー獲得）】

─────────────────────────────────────
① ランディングページ作成（ICE: 8.7）
─────────────────────────────────────
仮説：「アプリの価値を30秒で理解できれば、ログイン率は3倍以上になる」

実装内容：
  crane-analytics.vercel.app のトップに以下を追加

  ファーストビュー（スクロールなしで見える範囲）：
  ・キャッチコピー：「クレゲに使いすぎてない？投資額と期待値を即計算」
  ・サブコピー：「完全無料。Googleログインでデータをクラウド保存」
  ・アプリのスクリーンショット（プレイ中画面）
  ・「Googleで無料スタート」ボタン

  スクロール後：
  ・機能3点説明（損切りアラート・メルカリ相場・技術熟練度η）
  ・使い方GIF or スクリーンショット3枚
  ・「データは自分だけが見られる」セキュリティ説明

成功指標：ランディングページ→ログイン の遷移率 30%以上

─────────────────────────────────────
② OGP画像＋PWAアイコン設定（ICE: 8.3）
─────────────────────────────────────
仮説：「SNSでシェアしたときにリッチ表示されれば、クリック率2倍」

実装内容：

  <meta property="og:description" content="損切りラインを自動計算。メルカリ相場と比較してROIを最大化。">
  <meta property="og:image" content="https://crane-analytics.vercel.app/ogp.png">

  OGP画像（ogp.png）：
  ・1200×630px
  ・アプリのスクリーンショット＋タイトル＋キャッチコピー
